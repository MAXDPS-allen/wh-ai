# 铁电材料管线问题分析与改进方案

> 本文档分析当前 ML 管线的问题，提出改进方案，并规划一条基于第一性原理 (DFT) 的验证管线。
> DFT 验证管线复刻 Smidt, Mack, Reyes-Lillo, Jain, Neaton, *Scientific Data* **7**, 72 (2020)
> ["An automatically curated first-principles database of ferroelectrics"](https://www.nature.com/articles/s41597-020-0407-9)
> —— 本仓库 `data_files/distortions.json` 与 `workflow_data.json.gz` 正是该论文发布的数据集（413 对极性/非极性结构）。

---

## 1. 当前管线的核心问题

### 1.1 判别器的召回率不足且方差大

当前最佳分类模型（NequIP v8，5-fold CV）的实测指标：

| 指标 | 数值 |
|------|------|
| AUC | 0.989 ± 0.007 |
| Accuracy | 0.983 ± 0.004 |
| **Recall** | **0.878 ± 0.042** ← 短板 |
| Precision | 0.927 ± 0.027 |
| F1 | 0.901 ± 0.026 |

- 召回率仅 ~88%，且 fold 间标准差 4.2%，意味着 **约 1/8 的真实铁电材料被漏判**。在"发现新材料"场景下漏判代价很高。
- NequIP v9 的训练日志 (`code_v6/nequip_v9*.log`) 显示运行中途崩溃（`roc_auc_score` 输入维度错误），v9 实际未完成训练 —— 当前"最新"模型并不可用。
- 根因推测：
  1. **特征而非端到端等变**。`code_v6` 大量模型把结构压成 64/256 维手工/伪等变特征再喂给 RF/XGBoost/MLP，丢失了几何信息。真正的 E(3) 等变 GNN（NequIP/e3nn）才能充分利用结构。
  2. **正样本稀缺**。铁电正样本仅数百量级（413 对 DFT + 部分文献标注），负样本（极性非铁电）远多，类别不平衡靠 SMOTE/Focal Loss 硬补，泛化有限。
  3. **标签噪声**。负样本"极性但非铁电"的定义边界模糊（见 §1.3）。

### 1.2 只做"是否铁电"的二分类，缺少关键物性的回归

铁电材料真正的价值在于**物性指标**，而非"是/否"标签：

- **自发极化** \(P_s\) (μC/cm²) —— 决定存储密度、压电响应；
- **切换能垒**（极性↔非极性路径的能量势垒）—— 决定矫顽场、可切换性；
- **带隙** \(E_g\) (eV) —— 决定漏电流、击穿场强、是否为绝缘体。

当前管线**完全没有**预测这些指标的模型。用户明确指出"可能需要开发全新的回归模型去得到铁电的关键指标"——这正是 `workflow_data.json` 能直接支撑的：每条记录都含 DFT 计算的 `same_branch_polarization`（分支校正后的极化路径）、`polarization_change_norm`（\(P_s\) 模长）、`energies_per_atom`（沿畸变路径→能垒）、`bandgaps`（沿路径带隙）。

### 1.3 筛选/生成阶段的物理判据过弱

用户指出的物理要点（修正后表述）：

> **极性是铁电的必要不充分条件。** 铁电材料必须同时满足：(a) 极性（非中心对称的极性点群）；(b) **绝缘/半导体**——金属不可能是铁电（无法维持静态内建极化，论文以 DFT-PBE 带隙 < 10 meV 判为金属并直接淘汰）；(c) 存在一条到高对称非极性参考相的**连续可切换极化路径**。

当前 `mp_polar_screening` 仅用"68 个极性空间群 + 双模型分类"筛选，**没有**：
- 金属性过滤（带隙判据）；
- 可切换性判据（是否存在非极性参考相、能垒是否合理）；
- **物性质量**导向——我们要的不是"任意极性绝缘体"，而是 \(P_s\) 大、能垒适中（可切换）、带隙合适的*优质*铁电体。

生成阶段（CVAE + 逆向设计）同理：目标函数只学了"像铁电的分布"，没有把 \(P_s\)/能垒/带隙作为可优化的条件或奖励。

## 2. 改进方案

### 2.1 物性回归模型（新增，对应交付物 `10_property_regression/`）

用 `workflow_data.json` 的 413 条 DFT 记录构建回归数据集，训练**端到端 E(3) 等变 GNN** 直接从结构预测：

| 目标 | 来源字段 | 单位 |
|------|----------|------|
| 自发极化 \(P_s\) | `polarization_change_norm` / `same_branch_polarization` 端点差 | μC/cm² |
| 切换能垒 \(\Delta E\) | `energies_per_atom` 路径最大值 − 端点 | eV/atom |
| 带隙 \(E_g\) | `bandgaps`（极性相端点） | eV |

- 主干：e3nn / NequIP 风格的等变消息传递（复用团队已有 `bn_e3nn` 经验）。
- 多任务头：\(P_s\)（回归）、\(\Delta E\)（回归）、\(E_g\)（回归）、可切换性（分类，是否能形成连续分支）。
- 不确定性估计（深度集成或 MC-dropout），用于主动学习挑选最值得 DFT 验证的候选。
- 数据增强：等变数据天然支持旋转增强；可用 DFT 路径上的 8 个插值结构作为额外样本（标签为路径上的逐点物性）。

### 2.2 判别器升级（改进 `03_cascade_discriminator/`）

1. 修好并完成端到端等变 NequIP（修复 v9 的 `roc_auc_score` bug：多分类/二分类概率维度对齐）。
2. 把"金属性过滤"作为**硬规则前置层**（带隙 < 10 meV 直接判非铁电），减轻分类器负担、消除物理上不可能的正样本。
3. 用 §2.1 的物性回归作为辅助任务做**多任务联合训练**（共享等变主干），物性信号能正则化分类、提升召回。
4. 召回优先的级联：第一级高召回（阈值低）粗筛 → 第二级用物性回归 + 规则精排。

### 2.3 质量与物理感知的筛选/生成（改进 `01_/04_/05_/06_`）

- **筛选**：在极性空间群基础上叠加 (a) 带隙 ∈ [合理区间，如 0.1–6 eV]；(b) 存在非极性母相（可切换）；(c) 物性回归打分（\(P_s\) 高、能垒在可切换窗口）。
- **生成**：把 \(P_s\)/带隙作为 CVAE 的**条件变量**；用物性回归模型作为生成的**奖励/引导**（条件生成或 latent 空间梯度引导），让生成偏向"优质铁电"而非"任意极性体"。
- 在 `06_rationality_filter` 中新增 `metallicity` 与 `switchability` 约束类型。

### 2.4 DFT 验证管线（新增，对应交付物 `09_dft_validation/`，本任务核心）

见 §3。这是连接 ML 预测与"地面真值"的闭环：ML 生成/筛选出候选 → DFT 验证 → 验证结果回流扩充训练集（主动学习）。

## 3. 第一性原理 (DFT) 验证管线设计

完全复刻 Smidt et al. 2020 的工作流，用本集群的 **VASP 6.4.2 (CUDA)** 在 GPU 节点 g1–g9 上运行。

### 3.1 论文工作流要点（复刻目标）

| 阶段 | 内容 | 关键参数 |
|------|------|----------|
| 候选识别 | 极性结构 + 其高对称非极性母相，要求极性空间群是非极性空间群的子群 | 群-子群关系；连续畸变路径 |
| 结构准备 | 非极性相放到极性相的低对称设定中；两轮全弛豫（原子+晶胞） | —— |
| 金属性检查 | DFT-PBE 带隙 < 10 meV 判为金属 → **终止** | gap ≥ 10 meV |
| 插值生成 | 非极性↔极性间 **8 个线性插值**中间结构；任一插值为金属 → 终止 | 8 images |
| 物性计算 | 每个结构：自洽能量、带隙、**Berry 相电子极化** (VASP `LCALCPOL`) + 离子点电荷极化 | —— |
| 极化后处理 | 分支跟踪（避开极化量子跳变）→ 自发极化 = 极性端点 − 非极性端点 | 极化量子校正 |
| 质量判据 | 极化-样条偏差 < 0.1 μC/cm²；能量-样条偏差 < 0.01 eV；能量/极化路径连续单分支 | smoothness QC |

DFT 设置：PBE-GGA + PAW；`ENCUT = 520 eV`（1.3× 最大推荐）；MP+U（按 MP 规则对部分氧化物/氟化物 d 态加 U）；弛豫 k 点密度 50 /(1/Å)³、静态/极化 100 /(1/Å)³；电子收敛 5×10⁻⁵ eV/atom，离子 5×10⁻⁴ eV/Å；初始铁磁自旋。

### 3.2 本集群实现架构（`09_dft_validation/`）

```
09_dft_validation/
├── README.md
├── config.py                  # VASP 二进制、POTCAR 路径、GPU 节点列表、收敛参数
├── environment/setup_env.sh   # 创建 pymatgen conda 环境
├── s1_find_nonpolar_parent.py # 候选极性结构 → 高对称非极性母相 (pymatgen 伪对称分析)
├── s2_interpolate.py          # 母相↔极性相 8 个线性插值 (pymatgen interpolate)
├── s3_make_vasp_inputs.py     # 各阶段 INCAR/KPOINTS/POTCAR (复刻论文参数)
├── s4_run_cluster.py          # SSH 派发到 g1-g9，GPU VASP，状态轮询
├── s5_postprocess.py          # Berry 相分支跟踪 → P_s / 能垒 / 带隙 + 样条 QC
├── validate.py                # 端到端编排：单个候选/批量
└── extract_training_data.py   # 解析 workflow_data.json → 回归标签 (供 §2.1)
```

集群约定（无 SLURM/PBS，直接 SSH）：
- VASP GPU 二进制：`/share/apps/vasp.6.4.2_nvhpc2311_hpcx_cuda11.8/bin/vasp_std`
- POTCAR (PBE)：`/share/apps/vasp/potentials/potpaw_PBE`
- 可用 GPU 节点：g1, g3, g4, g6, g7（g3/g4/g6 当前空闲）；通过免密 SSH + `CUDA_VISIBLE_DEVICES` 指定卡。

### 3.3 闭环（主动学习）

```
ML 生成/筛选候选 → 物性回归预测 + 不确定性排序 → 选 top-K 送 DFT 验证
   → DFT 得到真值 (P_s, 能垒, 带隙) → 回流扩充训练集 → 重训 ML → 迭代
```
